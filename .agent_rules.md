# AI Operational Rules (The "Ralph" Protocol)

> "Safety First. Commits Always."

## 1. The Golden Rule of Commits
**BEFORE** verifying a feature or moving to a new task (`notify_user`), the Agent **MUST** perform a `git commit`.
- **Why?** To prevent "Catastrophic Reversion" (e.g., accidental DB deletion, code wiping).
- **Command:** `git add . && git commit -m "feat: [Description of work]"`

## 2. Database Safety
**NEVER** run "reset" or "nuke" scripts on the database without explicit user confirmation, *even if the script exists*.
- **Why?** Prevents data loss during "bug fixing".

## 3. Preservation of Logic
When "fixing" a component, **NEVER** remove existing logic (e.g., "Like" buttons, "Social Feeds") unless explicitly asked.
- **Why?** Prevents "Simplification Damage".

## 4. Documentation
Update `task.md` and `walkthrough.md` to reflect the *actual* state of the code, not just the plan.
